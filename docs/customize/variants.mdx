# Variants

Components can be customized by adding styles in the `$theme.variants` section.  

```scss
@use "siimple" with (
    $variants: (
        "button": (
            "default": (
                "background-color": "primary",
                "color": "white",
            ),
            "secondary": (
                "background-color": "secondary",
            ),
        ),
    ),
);
```

### Default variant

In the `default` field you can provide the styles that will be applied to the default component styles. For example, all styles defined in `variants.button.default` will be applied to the `siimple-btn` class name: 

```scss
@use "siimple" with (
    $variants: (
        "button": (
            "default": (
                "background-color": "primary",
                "color": "white",
            ),
        ),
    ),
);
```

Will generate: 

```css
.siimple-btn {
    background-color: #33333;
    color: white;
    /* Other default button styles */
}
```

### Custom variants

Any other field will generate an additional class that can be only applied to the corresponding component, with the class `.is-{variant_name}`. For example, this variant:

```scss
@use "siimple" with (
    $variants: (
        "button": (
            "secondary": (
                "background-color": #000000,
            ),
        ),
    ),
);
```

Will generate:

```css
.siimple-btn.is-secondary {
    background-color: #000000;
}
```
